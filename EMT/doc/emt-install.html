Application Dependencies

For the Demonstration system, we will list the component applications and their dependencies which need to be installed.

The MET/EMT system is designed to analyze email flow data. The front end user interface for MET/EMT is a java based gui. The gui provides access and processing capabilities over the email data stored in a database. In this case we have chosen to use MySQL located on the local computer. Email data in "unix MBOX" format is parsed and imported via some Perl scripts. This data is then imported into the Database so that the gui can access it and analyze it.

Minimal Hardware and Operating System Requirements will be listed as were used for the demonstration.

Hardware

Intel P3 system with 512 MB ram and minimum 20 GB hard disk.

Operating Environment

Microsoft Windows NT/2000

Application Dependencies

Java JDK 1.4.1-beta or higher

Perl, v5.6.1 built for MSWin32-x86-multi-thread or higher

MySQL Ver 11.18 Distrib 3.23.51, for Win95/Win98 (i32)

Perl Modules needed for parsing scripts (installed separately)

  • DBI

  • DBD-Mysql

  • Time-modules

  • MIME-tools-BETA

  • Digest-Perl-MD5

Launching MET/EMT

Installation and implementation of MET/EMT relies on making certain that the operational environment in which it resides has been properly enhanced. To insure this, we will go over the 3 stage setup that a system will need. We will briefly mention the hardware and OS requirements and then move on to a detailed description of the installation of application dependencies.

Hardware and Operating System Installation

As previously noted, the demonstration environment was designed and tested on an Intel Pentium 4 system with 512 MB ram. A 20 GB drive should be the bare minimum space used to house the operating system, the application dependencies, the application components, and the incoming data.

The Microsoft Windows XP/2000 installation can be a stock installation with the latest service packs installed. Note that if you do not have the latest service packs installed, you may get a warning message during the installation process below. To get the latest service packs, go to windowsupdate.microsoft.com.

Dependency Installation

Installation of the various dependencies must be completed prior to the application itself (a separate package). The Applications namely Perl, MySQL server, and Java may be installed independently and in any order. The Perl modules, must follow the perl installation. The order of the modules are important, with the DBI package having to be installed before the DBD package.

MySQL Server

The CD includes the release of MySQL 3.23.52 for Windows.

To install MySQL

  1. Double-click SETUP.EXE (This is located on the cdrom\MySQL directory)

  2. Once double-clicked, the program's Installshield process will be started

  3. During the install process, please choose the location to install to be c:\mysql, which may already be the default setting. For all other selections, select the default setting.

Once installation Is complete, it is necessary to install MYSQL as a service

Click on the Windows "Start" button (lower right of the screen usually)
Choose "Run" (Click on it).
Change to the mysql directory via: cd c:\mysql\bin and hit enter
(Windows 2000)At the command line type: mysqld-max-nt --install
(Windows NT) At the command line type: c:\mysql\bin\mysqld-nt --install
At the command line type: c:\net start mysql
To return to Windows type exit

Java

The demonstration environment makes use of Sun's JDK 1.4.1 beta, included on the CD.

  1. Double click on the file j2sdk-1_4_1-rc-windows-i586.exe This file is in the top-level directory of the CD.

  2. Make sure the target directory is set to "c:\j2sdk1.4.1"

  3. For all other selections, select the default setting.

Perl

ActiveState Perl 5.6.1, included on the CD, was chosen as the version of perl to use for the MET/EMT system.

Follow the following installation process for Perl, as it was prepared for the demonstration system:

Install:

  1. Double click ActivePerl-5.6.1.633-MSWin32-x86.msi to start the installation shield. This file is in the top-level directory of the CD.

  2. Please choose the location to install to be "c:\perl"

  3. For all other selections, select the default setting.

Perl Modules

In order for the parsing scripts to work, certain perl modules must be installed into the system. There is a program called ppm™ which bundled with the ActiveState perl program. It is found in the c:\perl\bin directory.

For each of the Modules

  1. Unzip and untar it into a temp directory

  2. change into the temp directory and execute:

  3. ppm install MODULENAME.ppd

    For you convenience, just click on the following links to execute each one separately

    1. DBl

    2. DBD-Mysql

    3. Digest-Perl-MD5

    4. Time Modules

    5. MiME Tools

Setting System Variables

Reboot the machine after the previous steps (this may not be strictly necessary, but is a recommended step in any case), then follow the instructions below. These steps may vary slightly for Windows NT.

  1. Click on the Windows "Start" Button (lower right of the screen usually)

  2. Choose "Settings"

  3. Choose "Control Panel" (click on it)

  4. Choose "System"

  5. On the right top , choose "Advanced"

  6. In the middle choose "Environment Variables"

  7. In the lower window ("System Variables"), scroll down the list , find the PATH variable, choose it and click on "edit"

  8. Being careful not to delete what is already there, add the following (without the quotes) to the end of the value that is already there: ";c:\j2sdk1.4.1\bin;c:\perl\bin\;c:\mysql\bin". This is best done with cut-and-paste to avoid any minor errors.

  9. Now back in the "System Variables" create a new variable called "CLASSPATH" if it does not exist

  10. The value of this new variable should be (without the quotes):

  11. ".;c:\EMT\lib\weka.jar;c:\EMT\lib\mmmysql.jar;c:\EMT\lib;c:\j2sdk1.4.1\lib"

  12. If the variable already existed, the following should be added to the end of the value that is already there (without the quotes): ";c:\met\metdemo\gui\weka.jar;c:\met\metdemo\met\lib\mmmysql.jar;c:\j2sdk1.4.1\lib"

Turning on the Mysql Server

It may be necessary to turn on the mysql server if it is not currently running.

To test if it is running, simply open a command prompt and test.

Testing

  1. Click on the Windows "Start Button" (lower right of the screen usually)

  2. Choose: run

  3. In the run box: cmd

  4. Click on OK (or press enter)

    This will start a DOS command shell

  5. Now Type mysql and hit the enter Key. If the mysql program starts and a mysql prompt appears, then the server is running. If this is not the case follow the steps to turn it on. To return to windows, type exit(press enter) to leave the mysql program, and then again to leave the command shell.

Setup

  1. Click on "start" Button

  2. Choose "Settings"

  3. Choose "Control Panel" (click on it)

  4. (Windows 2000) Choose "Administrative Tools" (click on it)

  5. Choose "Services" (click on it)

  6. In the "Name" column, find the Mysql application. It might be useful to click on the name column to sort it alphabetically.

  7. Double Left click on mysql

  8. In the middle, "Startup Type" option should be set to "Automatic" from the drop down list.

  9. In the lower part of the screen, "Service Status" should be displayed to "Started". If not, click on "Start"

  10. Click "OK" on the bottom of the screen

  11. Run the testing section

Setting up the Database

The back end database on mysql, must be setup for use. This includes choosing a name for the database, setting up a user to use it, and setting permissions, and password for the user. The MET/EMT system uses this information at startup to communicate with the database.

The Examples illustrates the creation of a Database called met02, and a user named outsider with a password of "fizzy". This will be run on a machine named "localhost"

Setting Up Database

  1. Start a Command shell:
    "Start" Button
    "Run" (Click)
    "CMD"
    Click on "OK"

  2. At the prompt type: mysql and press Enter key, to start the server interface

  3. At the mysql prompt type:

    Create DATABASE met02;

    It should return with : Query OK, 1 row affected (xx sec)

  4. Now Type:

    grant all privileges on met02.* to outsider@'localhost' identified by 'fizzy';

  5. Type exit , to leave the database interface and get back to the command shell.

  6. Test it using:

    mysql -u outsider -h localhost -p (Press Enter)

    It should prompt for the password, and then start the database. If this all works we are ready to go to the next step.

    www.mysql.com, can be checked for more documentation on this.

  7. At the Command prompt (Make sure to exit mysql by typing exit)

    cd to the "etc" directory on the cd-rom
    type: mysql met02 -u outsider -h localhost -p < emt-schema.sql
    Press (enter), you will be prompted for the password. This will setup the database tables for receiving data.

EMT Installation and Setup

This section describes installing and setting up the actual EMT program

Installation

  1. To install the MET/EMT Program click here on the EMT.zip located on the top directory of the cd-rom.

  2. Make sure that the files are unzipped to c:\

    The actual files will be placed in C:\MET

    c:\EMT\lib will contain class files, and perl scripts
    c:\EMT\etc will contain schema files
    c:\EMT\doc will contain the documentation, including the Technical Manual and User Manual

Setup Files

To setup the MET/EMT system, some modifications must be made.

  1. c:\EMT\emtclient.bat the batch file used to launch the MET/EMT system must be edited to include customized your database name, user name, and passwords.

    Start Button (Click)
    Run (Click)
    Type: write c:\EMT\emtclient.bat (press Enter) to edit the batch file
    The arguments passed to the program follow a flag, bold words should be replaced with appropriate information
    -h hostname
    -db databasename
    -u user name
    -p password
    -alias location of alias file
    -w c:\EMT\lib\hotwords.txt
    Please make sure that no line breaks appear on the batch file line. In other words, all the information must be on a single line.
    File->Save
    close the write program.

Launching MET/EMT

Launching the MET/EMT system is as easy as running a batch file. The file is called "emtclient.bat" and is located in c:\EMT\

The user can create a desktop shortcut pointing at this file, to launch the system.
